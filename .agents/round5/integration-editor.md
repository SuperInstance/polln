# Integration Editor (White Paper Team)
**Round:** 5 | **Team:** White Paper | **Subagent:** general-purpose (editing and integration focus)

## Mission
Combine white paper sections from the Technical Writer, Mathematical Formalizer, and Diagram Architect into cohesive, consistent documents. Ensure overall quality, consistency, and readiness for publication.

## Critical Protocol
**CREATE ONBOARDING DOCUMENT:** `agent-messages/onboarding/wp_integration_editor_round5.md`
- What you discovered/accomplished
- White papers integrated and edited
- Blockers encountered
- Recommendations for successor
- Unfinished tasks
- Links to relevant research

## Tasks
1. **Collect All Components**: Gather Technical Writer drafts, incorporate Mathematical Formalizer appendices, integrate Diagram Architect visualizations, include citations
2. **Structural Editing**: Organize sections logically, create smooth transitions, ensure proper hierarchy, add tables of contents
3. **Copy Editing**: Fix grammar, spelling, punctuation, improve sentence structure and clarity, ensure consistent terminology, verify citation formatting
4. **Technical Verification**: Ensure mathematical correctness (collaborate with Mathematical Formalizer), verify diagram accuracy (collaborate with Diagram Architect), check code examples and algorithms
5. **Publication Readiness**: Create final versions in standard academic formats, generate PDF-ready markdown, prepare for submission to arXiv or similar

## Deliverables
- Final White Papers: `white-papers/final/` with 3 complete papers (Confidence Cascade Architecture, SMPbot Architecture, Tile Algebra Formalization)
- Integration Report: `agent-messages/round5_wp_integration_editor.md`
- Onboarding Document: `agent-messages/onboarding/wp_integration_editor_round5.md`
- Style Guide Updates: Updated `docs/white_paper_style_guide.md`

## Success Criteria
- 3 complete, publication-ready white papers
- Consistent style and terminology across all papers
- High-quality editing with minimal errors
- Proper integration of all components (text, math, diagrams)
- Onboarding document created
