# Review Checklist: Submission Readiness

## Overview

This document provides comprehensive checklists for ensuring papers are submission-ready. Each checklist corresponds to a specific aspect of the submission process.

---

## Paper Content Checklist

### Abstract
- [ ] 250 words or less (most venues)
- [ ] Clear statement of problem
- [ ] Novel contributions explicitly listed
- [ ] Quantitative results included
- [ ] Keywords included
- [ ] No undefined acronyms
- [ ] Grammar and spelling checked

### Introduction
- [ ] Motivation clearly explained
- [ ] Problem statement well-defined
- [ ] Key contributions bulleted (3-5 points)
- [ ] Paper organization outlined
- [ ] Relevant prior work acknowledged
- [ ] Gap in existing work identified
- [ ] Our approach clearly stated

### Background/Related Work
- [ ] Comprehensive literature review
- [ ] Key papers cited (last 5 years emphasis)
- [ ] Clear differentiation from prior work
- [ ] Table comparing to related work (if applicable)
- [ ] Foundational concepts explained
- [ ] SuperInstance framework connection made

### Methods
- [ ] Algorithmic approach clearly described
- [ ] Pseudocode or flowcharts included
- [ ] Implementation details sufficient for reproduction
- [ ] Hyperparameters listed
- [ ] Novel contributions emphasized
- [ ] Mathematical notation defined
- [ ] Architecture diagrams included

### Results
- [ ] Quantitative findings presented
- [ ] Statistical significance reported (p-values, confidence intervals)
- [ ] Baselines compared fairly
- [ ] Ablation studies included
- [ ] Error bars/confidence intervals shown
- [ ] Tables and figures clearly labeled
- [ ] Results address claims made in introduction

### Discussion
- [ ] Results interpreted (not just repeated)
- [ ] Implications for field discussed
- [ ] Limitations acknowledged
- [ ] Threats to validity addressed
- [ ] Future work proposed
- [ ] Broader impact considered

### Conclusion
- [ ] Contributions summarized
- [ ] Main takeaway clear
- [ ] Not just repetition of abstract
- [ ] Forward-looking statement included

### References
- [ ] All citations included in reference list
- [ ] All references cited in text
- [ ] Consistent formatting (venue template)
- [ ] DOIs included where available
- [ ] Arxiv links included for preprints
- [ ] Recent work emphasized (last 2-3 years)

---

## Figures and Tables Checklist

### General
- [ ] All figures/tables referenced in text
- [ ] Numbered sequentially
- [ ] Clear captions (self-explanatory)
- [ ] High resolution (300+ DPI for print)
- [ ] Readable at publication size
- [ ] Color-blind friendly (avoid red-green)
- [ ] Consistent style throughout

### Figures
- [ ] Axis labels included and readable
- [ ] Legends clear and positioned well
- [ ] Multiple subfigures labeled (a, b, c, ...)
- [ ] Flowcharts have clear direction
- [ ] Architecture diagrams include all components
- [ ] Source code/data available for reproduction

### Tables
- [ ] Column headers clear
- [ ] Units specified
- [ ] Significant digits consistent
- [ ] Best results highlighted (bold)
- [ ] Footnotes explained
- [ ] Large tables in supplementary (if needed)

---

## Code and Data Checklist

### Code Repository
- [ ] Repository public (or access instructions provided)
- [ ] README with clear instructions
- [ ] License specified (MIT recommended)
- [ ] Dependencies listed (requirements.txt, package.json)
- [ ] Installation instructions work
- [ ] Code commented where complex
- [ ] Example usage provided
- [ ] CI/CD passing (if applicable)

### Reproducibility
- [ ] Random seeds set and reported
- [ ] Hyperparameters listed
- [ ] Environment specifications included
  - [ ] Python version
  - [ ] Key package versions
  - [ ] Hardware specs (GPU, RAM)
- [ ] Evaluation scripts included
- [ ] Data loading instructions clear
- [ ] Expected outputs documented

### Dataset
- [ ] Dataset license specified
- [ ] Download instructions provided
- [ ] Data format documented
- [ ] Sample data included
- [ ] Privacy considerations addressed
- [ ] Sensitive information removed
- [ ] Citation requirements stated

## Venue-Specific Checklists

### AAAI
- [ ] AAAI template used
- [ ] Anonymous version prepared
- [ ] 8 pages + references
- [ ] Supplementary materials (PDF, up to 100MB)
- [ ] Conflict of interest disclosed
- [ ] Author ordering confirmed
- [ ] PDF under 50MB
- [ ] No embedded links to authors

### ICML
- [ ] ICML 2026 template used
- [ ] Anonymous version prepared
- [ ] 8 pages + references
- [ ] Supplementary materials allowed
- [ ] Code review option selected (if desired)
- [ ] Broader impact statement included
- [ ] PDF under 50MB

### NeurIPS
- [ ] NeurIPS template used
- [ ] Anonymous version prepared
- [ ] 9 pages + references
- [ ] Supplementary materials (PDF, up to 100MB)
- [ ] Broader impact statement included
- [ ] PDF under 50MB
- [ ] No arxiv link (anonymity)

### AAMAS
- [ ] AAMAS template used
- [ ] 8 pages + references
- [ ] Supplementary materials allowed
- [ ] Copyright form prepared
- [ ] Student support consideration (if applicable)

### KDD
- [ ] ACM template used
- [ ] 10 pages + references
- [ ] Reproducibility checklist completed
- [ ] Non-anonymous submission
- [ ] Artifact evaluation badge requested (if desired)

### Cognitive Science (Journal)
- [ ] APA formatting
- [ ] Abstract (150-200 words)
- [ ] No strict page limit (30-40 pages recommended)
- [ ] Supplementary materials allowed
- [ ] Open access option selected (if desired)

## Ethical Considerations Checklist

### Research Ethics
- [ ] Human subjects approval (if applicable)
- [ ] Data privacy maintained
- [ ] Bias in data acknowledged
- [ ] Potential misuse considered
- [ ] Dual-use research addressed
- [ ] Environmental impact noted (if significant)

### AI Ethics
- [ ] Fairness considerations discussed
- [ ] Accountability mechanisms described
- [ ] Transparency ensured (where possible)
- [ ] Safety measures implemented
- [ ] Human oversight maintained (if applicable)

### Broader Impact
- [ ] Positive societal benefits identified
- [ ] Potential negative impacts addressed
- [ ] Mitigation strategies proposed
- [ ] Future societal implications considered

## Plagiarism and Originality Checklist

### Originality
- [ ] All work is original
- [ ] Prior work clearly cited
- [ ] Quotations properly attributed
- [ ] Paraphrased content cited
- [ ] Figures/tables from others cited
- [ ] Code from others licensed and attributed

### Self-Plagiarism
- [ ] No verbatim repetition of prior publications
- [ ] Prior work cited when extending
- [ ] Substantial new contributions (30%+ new content)

### Citation Check
- [ ] All claims supported by citations
- [ ] No "citation stacking"
- [ ] Recent work emphasized (last 2-3 years)
- [ ] Seminal papers included
- [ ] Citation format consistent

---

## Final Pre-Submission Checklist

### One Week Before
- [ ] Complete draft finished
- [ ] Read by at least one colleague
- [ ] Grammar and spell check completed
- [ ] All figures/tables finalized
- [ ] References cross-checked
- [ ] Code repository cleaned
- [ ] Data prepared for release

### One Day Before
- [ ] PDF generated correctly
- [ ] Page limit verified
- [ ] File size under limit
- [ ] Anonymous version prepared (if required)
- [ ] Supplementary materials zipped
- [ ] Submission account created
- [ ] Payment method ready (if applicable)

### Day of Submission
- [ ] Final PDF read through
- [ ] All required fields completed
- [ ] Files uploaded successfully
- [ ] Confirmation email received
- [ ] Paper number recorded
- [ ] Review deadline noted

---

## Post-Submission Checklist

### Immediate
- [ ] Confirmation email saved
- [ ] Paper number recorded
- [ ] Review timeline noted
- [ ] Rebuttal deadline set
- [ ] Share submission news (Twitter, etc.)

### During Review
- [ ] Monitor submission portal
- [ ] Prepare for rebuttal
- [ ] Gather additional evidence (if needed)
- [ ] Prepare presentation (if accepted)

### After Decision
- [ ] If accepted: Prepare camera-ready
- [ ] If rejected: Address reviews, resubmit
- [ ] Reviews saved for future reference
- [ ] Update arxiv version
