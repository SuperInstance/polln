## 1.1 Pre-Money Valuation Framework

### Methodology Justification

For pre-revenue semiconductor startups, traditional DCF analysis is inappropriate. We employ a **multi-method triangulation approach**:

1. **Venture Capital Method** — Based on projected exit value and required return
2. **Comparable Transaction Method** — Based on similar startup funding rounds
3. **Cost-to-Duplicate Method** — Based on NRE and IP development costs
4. **Scorecard Method** — Adjusting baseline for team, market, and technology factors

```
Target Exit Value (Year 4-5):     $150M (base case)
Required Investor Return:         5x (venture-stage semiconductor)
```

### LTV Calculation Methodology

```
LTV = (Average Revenue per Customer × Gross Margin) × Retention Rate
      ─────────────────────────────────────────────────────────────
                    Customer Churn Rate

For Maker Segment:
LTV = ($35 × 70% GM) × 0.85 retention / 0.15 churn
LTV = $24.5 × 0.85 / 0.15 = $138.8

With hardware refresh cycle (Year 3 upgrade):
LTV = $138.8 + ($60 upgrade × 50% probability × 70% GM) = $159.8
```

1. **Mask-locked architecture eliminates DRAM for weights** — primary memory cost avoided

3. **Volume scaling path** — margin improvement at scale
4. **No software licensing costs** — embedded model eliminates per-unit royalty

**Key Risk**: LPDDR4 pricing volatility (30-40% of COGS)
