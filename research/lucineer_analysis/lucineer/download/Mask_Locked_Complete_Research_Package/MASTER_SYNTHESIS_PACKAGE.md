## Technical Architecture Summary

### Core Innovation: Mask-Locked Weights

Traditional chips store weights in memory (DRAM/SRAM), requiring energy-intensive reads for each computation. Mask-locking encodes weights directly into metal interconnect layers, achieving:

- **Zero access latency** - weights always present at compute unit
- **Zero access energy** - no memory read operations
- **Infinite bandwidth** - no bus bottlenecks
- **Tradeoff** - changing weights requires new chip fabrication

### Quantization Strategy

| Precision | Quality Retention | Model Size (3B) | Hardware Implication |
|-----------|------------------|-----------------|---------------------|
| FP16 | 100% | 6 GB | Baseline |
| INT8 | 98-99% | 3 GB | Moderate complexity |
| INT4 | 95-97% | 1.5 GB | Target baseline |
| INT2 | 85-92% | 750 MB | Research stage |
| Ternary {-1,0,+1} | 93-96% | ~1 GB | **Recommended** |

### Process Node Selection: 28nm

**Rationale:**
- NRE costs: $2-4M (vs. $15-20M for 4nm)
- Mask set: $1-3M (vs. $10M+ for leading edge)
- Established PDKs and design services
- Sufficient density for 1-7B models
- Better availability, lower supply risk
