## TIER 1: FUNDAMENTAL PARADIGM SHIFTS

These change what AI *is*, not just what it *does*.

### 1. Confidence Flow Theory
**File:** `confidence-cascades.md` (42KB)

**The breakthrough:** Confidence isn't a score—it's a currency that flows through tile chains with mathematical precision.

```
Sequential tiles: Confidence MULTIPLIES (0.90 × 0.80 = 0.72)
Parallel tiles: Confidence AVERAGES (weighted by trust)
Conditional tiles: Confidence DEPENDS on path taken
```

**Why it matters:** You can see trust flow through the system. When a pipe bursts (low confidence), you see it immediately. No more "trust me" AI.

**Key insight:** The Three-Zone Model
- GREEN (0.90-1.00): Auto-proceed
- YELLOW (0.75-0.89): Human review
- RED (0.00-0.74): Stop and diagnose

### 2. Stigmergic Coordination
**File:** `emergent-swarm.md` (33KB)

**The breakthrough:** Tiles don't need a boss. They communicate through the spreadsheet itself—leaving "digital pheromones" in cells. Other tiles detect these traces and react. Complex behavior emerges from simple rules.

```
Tile A writes to Cell X →
Tile B reads Cell X →
Tile B modifies Cell Y →
Tile C reads Cell Y →
...swarm intelligence emerges
```

**Why it matters:** We're not building distributed AI. We're building AI that self-organizes like ant colonies, bee hives, and starling flocks.

**Key insight:** Five levels of emergence
1. Individual tile behavior
2. Pairwise tile coordination
3. Local pattern formation
4. Global structure emergence
5. Adaptive self-modification

---

### 3. The Composition Paradox
**File:** `tile-safety.md`

**The breakthrough:** Safe tiles don't always compose safely. Two individually safe tiles can create unsafe behavior when combined.

```
Tile A (Safe): "Round numbers to 2 decimals"
Tile B (Safe): "Multiply by 100"

A + B = Different results depending on order
= Potential financial disaster
```

**Why it matters:** Current AI safety research focuses on individual models. We're the first to systematically analyze how safety constraints propagate (or don't) through composition.

**Key insight:** Constraints naturally STRENGTHEN during composition, not weaken. Each tile can only restrict, never expand.

---

### 4. Tile Algebra: Formal Verification
**File:** `tile-algebra.md`

**The breakthrough:** Tiles form a rigorous algebraic structure—a category. We can PROVE things about AI behavior, not just test it.

```
Traditional: f(g(h(x))) → Hope it works → Test empirically
Tile Algebra: Tile ∘ Tile ∘ Tile → PROVE it works → Guaranteed correctness
```

**Why it matters:** Going from "we think this bridge won't collapse" to "we can prove this bridge can't collapse."

**Key insight:** Tiles satisfy algebraic laws:
- Associativity: (A ∘ B) ∘ C = A ∘ (B ∘ C)
- Identity: id ∘ A = A ∘ id = A
- Distributivity: A ∘ (B + C) = (A ∘ B) + (A ∘ C)

### 5. Cross-Modal Tile Composition
**File:** `cross-modal-tiles.md`

**The breakthrough:** Text tiles, image tiles, and audio tiles can work together because they share a "latent room" where meaning is the same regardless of modality.

```
Text: "medical scan shows anomaly" →
    [SHARED LATENT SPACE] →
Image: [Generates visualization]
```

**Why it matters:** Tiles pass MEANING, not just data. "Cat" in text and a picture of a cat are the same thing in the shared space.

**Key insight:** Hybrid embedding wins (256-dim shared + 512-dim modality-specific)
- Retrieval: 0.89
- Cross-modal alignment: 0.84
- Knowledge transfer: 0.78

### 6. Counterfactual Branching
**File:** `counterfactual-tiles.md` (27KB)

**The breakthrough:** Tiles can branch into parallel simulations, exploring "what if" scenarios WITHOUT committing to any of them.

```
Traditional: "What if revenue +10%?" → One number: $1.1M
SMP: "What if revenue +10%?" → 10,000 parallel simulations
    → Mean: $1.1M, 5th: $950K, 95th: $1.3M, Loss probability: 12%
```

**Why it matters:** Not scenario planning—quantum decision visualization. See all possible futures before choosing one.

**Key insight:** The Counterfactual Tree
- Root tile: Current state
- Branches: Alternative scenarios
- Leaves: Probability distributions
- Integration: Choose best path with confidence

---

### 7. Tile Memory & Cumulative Learning
**File:** `tile-memory.md`

**The breakthrough:** Tiles maintain state across executions. They learn from use, not just training.

```
Execution 1: Memory {} → Output: "Legitimate" → Memory: {seen: 1}
Execution 100: Memory {seen: 99, patterns: [...]} → Output: "FRAUD DETECTED"
```

**Why it matters:** Traditional AI is stateless. SMP tiles get smarter with every execution. No retraining required.

**Key insight:** L1-L4 Memory Hierarchy
- L1: Register (current execution)
- L2: Working (recent context)
- L3: Session (conversation state)
- L4: Long-term (persistent patterns)

---

### 8. Federated Tile Learning
**File:** `federated-tiles.md` (43KB)

**The breakthrough:** Organizations share LEARNED DECISION BOUNDARIES as inspectable tiles, not raw gradients.

```
Traditional FL: Share gradients (black box) → Blind aggregation
Federated Tiles: Share tiles (inspectable) → Validate before integration
```

**Why it matters:** Hospital A and Hospital B can collaborate without sharing patient data, without blind trust, and with full visibility into what's being learned.

**Key insight:** Adversarial resilience
- Malicious tiles are visible and rejectable
- No poisoning attacks through hidden gradients
- Each tile validated before integration

**File:** `distributed-tiles.md` (49KB)

**The breakthrough:** Tiles live wherever they need to be—laptop, AWS GPU, edge device at the factory—and work together like they're local.

```
┌──────┐      ┌──────────┐      ┌─────────┐
│Laptop│      │AWS GPU   │      │Edge Dev │
│Tile A│ ──── │Tile B    │ ──── │Tile C   │
└──────┘      └──────────┘      └─────────┘
    │              │                  │
    └──────────────┴──────────────────┘
             THEY TALK TO EACH OTHER
             Same spreadsheet. Different machines.
```

**Why it matters:** The spreadsheet makes distributed systems INVISIBLE. You don't think about which tile runs where. You just draw arrows.

### 11. KV-Cache Cell Sharing
**File:** `kv-cache-sharing.md`

**The breakthrough:** Multiple tiles share cached KV states. If Tile A computed something, Tile B can reuse it without recomputation.

**Why it matters:** Massive efficiency gains. Tiles don't repeat work they've already done.

**File:** `execution-strategies.md`

**The breakthrough:** Cells can automatically route to parallel/series, sync/async based on dependencies.

Right-click cell → "Parallel" → System generates execution plan
Same cells. Same formulas. Different execution. 15x faster.

**Why it matters:** Parallel programming isn't for experts anymore. Anyone who can use a spreadsheet can now build distributed systems.
